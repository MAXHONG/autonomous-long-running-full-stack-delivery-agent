```text

恢复要求：
1. 先检查当前目录、git 状态、最近提交和未提交改动。
2. 先读取 task.json、progress.md、architecture.md、handoff.md。
3. 执行最小启动路径和冒烟验证。
4. 如果环境损坏，先修环境。
5. 从 task.json 中选择一个高优先级、依赖满足、可验证的任务闭环继续推进。
6. 实现后立即测试，失败先修复并回归验证。
7. 每轮结束更新状态文件和必要文档。
```
